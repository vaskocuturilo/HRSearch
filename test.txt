Задача

    Написать на языкe Python SMTP-клиент и SMTP-сервер.
    
    IDE: JetBrains PyCharm Community Edition
    Версия IDE: последняя из доступных
    Версия Python: 2.7
    
    Приветствуется использование CVS(git или hg), если есть навыки. 
    Выкладывать куда-то не обязательно, достаточно локального репозитория.

Требования

    - Сервер
      - сервер должен обрабатывать максимально возможное кол-во одновременно подключенных клиентов 
      - сервер обслуживает клиента по протоколу SMTP, реализация протокола в минимальном виде, т.е. не надо
        поддержки опций, сложных логик и т.п., только прием писем
      - за одну SMTP-сессию сервер должен способен принять от клиента 1 или более писем
      - сервер должен корректно обрабатывать ошибки протокола и соединения
      - полученные письма сервер складывает в CSV-файл(колонки: IP, DATE, FROM, TO, SUBJECT, BODY)
      - сервер имеет собственный CLI
          команды: 
            start - запустить сервер
            stop - остановить сервер
            statistics - вывести статистику(что выводить на ваше усмотрение)
    - Клиент
      - клиент динамически генерирует поля письма для кажой посылки, т.е. FROM, TO, SUBJECT, BODY должны быть рандомные
      - клиент должен корректно обрабатывать ошибки протокола и соединения
      - клиент имеет собственный CLI
          команды:
            start - запустить клиент
            stop - остановить клиент
            statistics - вывести статистику(что выводить на ваше усмотрение)
      - клиент принимает 2 параметра командной строки: домен-FROM, домен-TO(например: python smtp-client from.example.com to.example.com)
      - поля FROM и TO генерируются на основе указанных доменов FROM/TO. т.е. если домен-FROM у нас идет как from.example.com, 
        то поле FROM письма может принять формы qwe@from.example.com, asd@from.example.com, xxx@from.example.com, 
        но не zxc@someotherdomain.com
